O que é a cloud (computação na nuvem)?

A cloud é usar servidores na internet para guardar ficheiros e usar aplicações, em vez de depender só dos computadores da escola. Isto é válido para Google, Microsoft, Amazon e outros. A vantagem é que podemos aceder de qualquer lugar, partilhar, e não ficamos dependentes de uma máquina.

Na escola isto traduz-se em: trabalhos no Classroom, documentos no Drive, formulários online, contas seguras — tudo isto é cloud.

1. Segurança na Cloud (Cloud Security)

ProfessoresEquipa TICEncarregados de educação

Usar cloud sem segurança é pôr a mochila no recreio aberta. Esta parte é para garantir que isso não acontece.

  • Ativar autenticação de dois fatores nas contas institucionais.
  • Partilhar documentos apenas com utilizadores da escola.
  • Explicar a pais e alunos que os dados não ficam “na internet”, ficam em serviços geridos.

2. Serviços na Cloud e Serviços Locais (Cloud vs On-Premises)

ProfessoresAlunosEquipa TIC

Nem tudo tem de estar na internet. Algumas coisas fazem mais sentido ficar dentro da escola. O importante é saber quando usar cada um.

  • Cloud: aceder em casa, no telemóvel, partilhar com a turma — Google Drive, OneDrive, Moodle alojado.
  • Local: coisas que só devem estar na escola ou que dependem da rede interna.
  • Híbrido: a realidade das escolas — parte na cloud, parte local.
Recursos para aprofundar
Microsoft – Cloud vs servidor local Wikipédia – Computação em nuvem YouTube – Servidor local vs Cloud (PT)

3. Infraestrutura como Código (Infrastructure as Code – IaC)

Professores TICAlunos cursos profissionaisEquipa TIC

É o lado “engenharia” da cloud: montar ambientes com ficheiros, não com cliques. Mostra para onde vai a área.

  • Criar ambientes de laboratório repetíveis para turmas.
  • Guardar a configuração como código e versionar.
  • Explicar aos alunos que infraestruturas também podem ser automatizadas.

4. Serverless (Computação sem servidor)

Professores TICAlunos

Queres pôr uma função a correr na cloud sem comprar servidor? É isto. Ideal para projetos pequenos de alunos.

5. Serviços Google na Cloud (Google Cloud Services)

ProfessoresAlunosEncarregados de educação

A maior parte das escolas já usa Google. Convém explicar que isso já é cloud e o que cada serviço faz.

  • Google Drive: guardar e partilhar ficheiros na nuvem.
  • Google Classroom: gerir turmas e trabalhos.
  • Google Docs/Sheets/Slides: trabalhar em conjunto no mesmo documento.
  • Google Sites: criar páginas dentro da plataforma. Não aceita HTML externo.
Recursos para aprofundar
Google Workspace for Education Ajuda oficial – Google Sites YouTube – Introdução ao Google Sites
Ver explicação técnica: HTML, iframe e Google Sites

1. HTML
HTML é a linguagem que define a estrutura de uma página web. Um ficheiro .html contém o texto e marcações que o navegador interpreta para mostrar a página.

2. iframe
Um <iframe> é uma janela dentro de uma página que mostra outra página. No Google Sites, quando usas “Incorporar → URL”, ele usa um iframe por baixo.

3. Porque o Google Sites não mostra HTML do Google Drive
O Drive serve ficheiros para download ou visualização, não como site. E o Sites bloqueia incorporar páginas com restrições de segurança. Resultado: o HTML do Drive não aparece renderizado.

4. Fluxo correto
Publicar o HTML num serviço que sirva páginas (GitHub Pages, Netlify, Vercel), obter o link público e incorporar esse link no Google Sites. Assim a tua página abre corretamente.

5. Motivo
O Google Sites é seguro por design. Bloquear HTML externo evita que alguém injete código malicioso.

6. Exemplos na Escola

ProfessoresAlunosEncarregados de educação

Algumas formas simples de usar cloud no agrupamento.

  • Publicar trabalhos de alunos em Google Sites (feito dentro do Sites).
  • Se o trabalho for um site HTML feito pelos alunos, publicar primeiro em GitHub Pages ou Netlify e só depois ligar no Google Sites.
  • Partilha de materiais de departamento no Drive.
  • Mostra de projetos digitais a encarregados de educação.
