Competências e Conhecimentos Cloud (Cloud Skills and Knowledge)

O que é a cloud (computação na nuvem)?

A cloud é usar servidores na internet para guardar ficheiros e correr aplicações, em vez de depender apenas do computador da escola. É uma ideia geral, válida para Google, Microsoft, Amazon ou qualquer outro serviço. Permite aceder aos mesmos ficheiros em casa, na escola ou no telemóvel.

Para a escola, isto significa colaboração, cópias de segurança automáticas e menos dependência de equipamentos locais. Cloud não é “pôr tudo na internet”, é usar a internet com segurança e responsabilidade.

1. Segurança na Cloud (Cloud Security)

ProfessoresEquipa TICEncarregados de educação

Garantir a segurança é o primeiro passo quando se usa cloud. A segurança envolve proteger dados de alunos e docentes, definir bem as permissões e promover boas práticas digitais.

  • Ativar autenticação de dois fatores para contas institucionais.
  • Usar partilha restrita a pessoas da escola.
  • Ensinar que o fornecedor protege a infraestrutura, mas a escola controla acessos e partilhas.
Recursos para aprofundar
NAU – Segurança na Cloud (PT) Microsoft – Segurança na Cloud

2. Serviços na Cloud e Locais (Cloud vs On-Premises)

ProfessoresAlunosEquipa TIC

Alguns serviços ficam na internet (cloud) e outros na escola (local). O importante é escolher o que faz sentido para cada caso.

  • Cloud: acesso de qualquer lugar e partilha fácil (ex.: Google Drive, OneDrive).
  • Local: servidores internos para dados mais sensíveis ou sem ligação à internet.
  • Híbrido: a combinação dos dois é o modelo mais comum em escolas.
Recursos para aprofundar
Microsoft – Cloud vs Servidor Local Wikipédia – Computação em Nuvem

3. Infraestrutura como Código (Infrastructure as Code – IaC)

Professores TICAlunos cursos profissionaisEquipa TIC

IaC é uma forma moderna de gerir servidores e redes na cloud. Em vez de configurar tudo à mão, define-se tudo em ficheiros, que podem ser reutilizados e testados.

  • Descrever num ficheiro o ambiente que a cloud deve criar.
  • Garantir consistência entre anos letivos ou turmas diferentes.
  • Reduzir erros e facilitar aprendizagem prática de DevOps.
Recursos para aprofundar
AWS – O que é IaC? Google Cloud – IaC OVHcloud – Infraestrutura como Código

4. Serverless (Computação sem Servidor)

Professores TICAlunos

Serverless significa correr código sem se preocupar com servidores. A cloud trata de tudo. É útil em projetos simples e educativos.

  • Automatizar tarefas, como enviar emails quando um formulário é submetido.
  • Executar aplicações de alunos sem configurar infraestrutura.
  • Reduzir custos – só se paga quando o código é executado.
Recursos para aprofundar
IBM – Computação sem Servidor Red Hat – Serverless Binario Cloud – Serverless e como aplicar

5. Serviços Google na Cloud (Google Cloud Services)

ProfessoresAlunosEncarregados de educação

Os serviços Google são exemplos claros de cloud na escola. Não requerem instalação e funcionam em qualquer dispositivo com internet.

  • Google Drive: armazenar e partilhar ficheiros.
  • Google Docs / Sheets / Slides: editar documentos em grupo.
  • Google Classroom: gerir turmas e enviar trabalhos.
  • Google Sites: criar páginas dentro da plataforma — não permite enviar HTML externo.
Recursos para aprofundar
Google Workspace for Education Ajuda Google Sites
